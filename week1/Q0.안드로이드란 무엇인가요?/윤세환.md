## 안드로이드 아키텍쳐
- Linux Kernel
  - 메모리, 프로세스, 저장소, 카메라 그리고 각종 센서와 같은 하드웨어 그리고 화면, 오디오와 같은 몇몇 하드웨어 장치를 구동하는데 필요한 드리아브들을 관리한다.
- Hardware Abstract Layer
  - 하드웨어 기능을 인터페이스로 제공한다.
- Native C/C++ Libraries
  - C/C++로 작성된 라이브러리의 모음으로, 성능 집약적인 작업을 위해 사용되며 그 예시로는 그래픽 랜더링을 위한 OpenGl, 로컬 데이터 저장을 위한 SQLite, 웹 페이지를 표시하는데 사용되는 WebKit등이 존재한다.
- Android Runtime (ART)
  - Kotlin이나 Java에서 컴파일된 바이트코드를 사용해 앱을 실행하며, 최적화된 성능을 위해 AOT와 JIT 컴파일 방식을 지원한다.
- Android Framework
  - ActivityManager, NotificaionManager와 같은 앱 개발을 위한 고수준 API를 제공하며, 이를 통해 개발자가 안드로이드 시스템 기능을 효율적으로 사용할 수 있도록 한다.
- Application
  - 시스템 앱 및 플레이스트어에서 받은 앱과 같이 사용자에게 직접적으로 다양한 기능을 제공하는 소프트웨어

## Q 안드로이드 플랫폼 아키텍처는 Linux Kernel, ART, HAL 등 여러 계층으로 구성됩니다. 이 구성 요소들이 애플리케이션 실행과 하드웨어와의 상호 작용을 위해 어떻게 작동하는지 설명해 주세요.
- 하드웨어 기능을 제공하는 어플리케이션을 위해 개발자는 Android Framework를 사용하여 Application을 구현합니다.
- Android Framework는 안드로이드 시스템에 대한 고수준 API를 제공하며 바인터 IPC를 통해 시스템 서비스에 요청을 전달합니다.
- ART에서 실행중인 시스템 서비스에서 JNI를 통해 C/C++ Native Libraries를 호출합니다.
- C/C++ Native Libraries에서 하드웨어 접근이 필요한 기능의 경우 HAL 인터페이스를 호출합니다.
- HAL에서 실제 Linux Kernel에 포함된 하드웨어 드라이버를 호출하여 하드웨어와 상호작용이 이루어집니다.
