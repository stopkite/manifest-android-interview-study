##  AndroidManifest.xml이란
- 앱에 대한 필수 정보가 담긴 구성 파일로 안드로이드 운영 체제, 빌드 도구, 플레이스토어에 제공하기 위해 존재
- 앱의 구성 요소(Activity, Service, BroadcastReceiver, ContentProvider), 앱이 필요로 하는 권한과 하드웨어 및 소프트웨어 기능 등을 정의

## AndroidManifest.xml의 주요 기능
1. 앱의 구성 요소 선언
  - Activity, Service, BroadcastReceiver, ContentProvider를 등록하여 안드로이드 시스템이 이를 시작하거나 상호작용하는 방법을 알 수 있도록 함
2. 앱에서 사용하는 권한 선언
  - 인터넷, 파일 접근, 위치, 블루투스와 같이 앱에서 사용하는 권한을 선언함
  - 이를 통해 사용자로 하여금 이 앱이 어떤 권한을 사용하는지 할고 이를 승인/거부할 수 있도록 함
3. 하드웨어 및 소트프웨어 요구 사항
  - 카메라 GPS 또는 특정 화면 크기와 같이 앱이 필요로 하는 기능을 정의함으로써 플레이스토어에서 요구사항을 충족하는 기기만 설치가 가능하도록 함
4. 앱 메타 정보
  - 앱의 패키지 이름, 버전, 최소 및 대상 API 레벨, 테마, 스타일과 같은 정보를 제공
  - 단, 패키지 이름, 버전, 최소 및 대상 API 레벨과 같은 속성값의 경우 build.gradle에 정의된 값으로 덮어씌워지므로 build.gradle과 같은 빌드 파일에 해당 내용을 정의한 경우에는 불필요
5. 인텐트 필터
  - 앱의 구성 요소가 처리할 수 있는 Intent 종류를 명시함으로써 다른 앱과 적합한 상호작용을 수행할 수 있도록 함
6. 앱 구성 및 세팅
  - 메인 런처 Activity 정의, 백업 동작 구성, 테마 지정과 같은 구성을 선언

## Q. AndroidManifest의 인텐트 필터는 앱 상호 작용을 어떻게 가능하고 하고, 액티비티 클래스가 AndroidManifest에 등록되어 있지 않으면 어떻게 되나요?
-  다른 앱과의 상호작용을 위해 암시적 인텐트를 사용해 해당 인텐트를 처리할 수 있는 구성 요소를 호출하게 되며, 안드로이드 시스템에서 이 암시적 인텐트를 처리할 수 있는 구성 요옷를 각 앱의 Android Manifest 파일을 참고하여 분석한 후 해당 요소를 호출하여 타 앱간의 상호작용을 가능하게 함
- 만약 이 과정에서 해당 인텐트를 처리하는 구성 요소를 찾지 못하면 `ActivityNotFoundException` 예외가 발생
- 액티비티 클래스가 AndroidManifest에 등록되어 있지 않다면 앱을 실행하는 intent를 처리할 수 없는 액티비티를 찾지 못하게 되므로 역시 `ActivityNotFoundException`이 발생
