## Activity의 생명주기
1. onCreate
- Activity가 생성될 때 호출되는 첫 번째 메서드로, Activity의 생애 주기동안 한 번만 호출된다.
- Activity에서 사용하는 객체를 초기화하고, setContentView와 같은 메서드를 토해 UI 컴포넌트를 설정하며 savedInstanceState을 사용하여 Configuration Change와 같은 상황으로 인해 Activity가 파괴된 후 복구되었을 시 데이터를 복구하는 기능을 수행

2. onStart
- Activity가 사용자에게 표시되고 있으나 상호작용은 불가능한 상태

3. onRestart
- onStop된 Activity가 다시 사용자에게 표시되는 경우 onStart전에 호출

4. onResume
- Activity가 foreground에 존재하며 사용자와 상호작용할 수 있는 상태

5. onPause
- Activity가 사용자아게 표시되고 있으나 다이얼로그와 같은 다른 요소로 인해 부분적으로 가려질 때 호출
- 애니메이션, 센서 업데이트 또는 데이터 저장과 같은 작업을 일시 중지할 때 사용

6. onStop
- Activity가 더 이상 보여지지 않을 때 호출

7. onDestroy
- Activity가 완전히 소멸되고 메모리에서 제거되기 전 호출
- 개발자가 직접 해제가 필요한 리소스에 대해 해제하는 작업을 수행해야 함

## Q. onPause()와 onStop()의 차이점은 무엇인지 설명하고, 리소스 점유율이 높은 작업을 처리하는 경우 해당 메서드들을 어떤 시나리오에서 사용해야 하나요?
- onPause의 경우 Activity가 사용자에게 보이지만 다이얼로그와 같은 다른 UI로 인해 부분적으로 가려져 보일 때 호출
- onStop의 경우 Activity가 사용자에게 보여지지 않을 떄 호출
- onPause에서는 사용자 상호작용과 관련된 리소스를 중단해야 함
  - 동영상 플레이어 일시 중단
  - 카메라 Preview 중단
- onStop에서는 앱이 보이지 않을 때 더 이상 필요없는 무거운 리로스를 해제해야 함
  - Activity에서 특정 데이터 flow를 collect하여 UI를 갱신하는 로직이 있다면 이를 중단해야 함

## Activity A에서 Activity B를 시작하고, 다시 돌아왔을 때의 생명주기 순서
1. Activity A에서 B로 이동하는 경우
  - Activity A에서 onPause 호출
  - Activity B에서 onCreate -> onStart -> onResume 호출
  - Activity A에서 onStop 호출
2. Activity B에서 A로 돌아오는 경우
  - Activity B에서 onPause 호출
  - Activity A에서 onRestart -> onStart -> onResume 호출
  - Activity B에서 onStop -> onDestroy 호출

## Activity의 lifecycle 인스턴스란 무엇인가요?
- 모든 Activity가 가지고 있는 객체로, Activity의 생명주기 이벤트를 관찰하고 이에 반응하는 방법을 제공한다.
- Activity의 현재 생명주기 상태를 나타내며 onCreate, onStart, onResume과 같은 생명주기 이벤트를 직접 재정의하지 않고 관찰할 수 있는 방법을 제공
- 이를 통해 Activity 생명주기에 맞추어 동작해야 하는 별도의 클래스가 있을 때 해당 클래스의 메서드를 Activity의 onCreate, onStart, onResume 등에 직접 호출하지 않고 독립적으로 관리할 수 있도록 함

''' kotlin
class LoggingObserver(logService: LogService) : DefaultLifecycleObserver {
    override fun onStart(owner: LifecycleOwner) {
        super.onStart(owner)
        logService.log("onStart detected")
    }

    override fun onStop(owner: LifecycleOwner) {
        super.onStop(owner)
        logService.log("onStop detected")
    }
}

class MainActivity : ComponentActivity() {
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        lifecycle.addObserver(LoggingObserver(LogServiceImpl()))
    }
}
'''
