### BroadcastReceiver란
- 안드로이드 운영 체제나 앱 자체적으로 발생한 브로드 캐스트 메세지를 수신하고 처리하는 컴포넌트
- Activity, Service의 생명주기에 직집적으로 연결되지 않을 수 있는 이벤트를 처리하는 데 사용되며, 앱이 백그라운드에서 계속 특정 변경사항을 감지하지 않고도 해당 변경사항에 반응할 수 있도록 하여 리소스를 절약

### Broadcast 유형
- 시스템 브로드캐스트
    - 안드로이드 운영 체제에서 발생한 브로드캐스트 신호로
    배터리 잔량 변경, 시간대 업데이트 또는 네트워크 연결 변경과 같은 이벤트를 보낸다.
- 커스텀 브로드캐스트
    - 앱 내부 혹은 앱 간의 특정 정보나 이벤트를 전달하기 위해 보내는 이벤트

### BroadcastReceiver 등록
1. Manifest 파일에 등록
    - 앱이 실행중이지 않을 때도 처리해야 하는 이벤트에 사용
    - AndroidManifest.xml 파일에 receiver를 통해 BroadcastReceiver 클래스를 등록하고, 해당 컴포넌트 태그 내에 intent-filter를 설정하여 어떤 메세지를 수신할 지 설정
2. 런타임 동적 등록
    - 앱의 특정 상황에서만 처리해야 하는 이벤트에 사용
    - registerReceiver를 통해 BroadcastReceiver 객체를 등록

### 유의 사항
- 런타임 동적 등록을 통해 BroadcastReceiver를 등록한 경우에는 반드시 메모리 누수를 방지하기 위해 unregisterReceiver를 사용하여 리시버를 해제해야 한다.
- API 레벨 26 이후부터 백그라운드 앱은 브로드캐스트 수신에 제약을 받으므로, 이 경우에는 Context.registerReceiver 또는 JobScheduler를 사용해야 한다.

### Q. 브로드캐스트의 유형에는 어떤 것이 있으며, 기능 및 사용 측면에서 시스템 브로드캐스트와 커스텀 브로드캐스트는 어떤 차이가 있나요?
- 브로드캐스트는 시스템/커스텀 브로드캐스트가 존재한다.
- 시스템 브로드캐스트의 경우 안드로이드 시스템에서 발생한 메세지를 처리하기 위해 사용되는 반면 커스텀 브로드캐스트는 앱에서 직접 생성한 커스틈 메세지를 처리하기 위해 사용된다.
