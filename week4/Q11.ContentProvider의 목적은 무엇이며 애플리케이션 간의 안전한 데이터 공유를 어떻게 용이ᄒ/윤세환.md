### ContentProvider란
- 구조화된 데이터에 대한 접근을 관리하고 앱간 데이터 공유를 위한 표준화된 인터페이스를 제공하는 컴포넌트
- 데이터 접근 로직을 캡슐화하여 앱 간 데이터 공유를 더 쉽고 안전하게 만드는 것이 목적
- 데이터 접근 주소로 URI를 사용

### ContentProvider의 URI 구성 요소
`content://com.example.contacts/contacts`
- 권한 : ContentProvider를 식별
- Path : 접근하려는 데이터 유형을 지정
- ID (선택) : 데이터 셋 내의 특정 항목 지정

### ContentProvider의 메서드
- query() : 데이터 검색
- insert() : 데이터 추가
- update() : 기존 데이터 수정
- delete() : 데이터 제거
- getType() : 데이터의 MIME 타입 리턴
- getStreamType(): 파일의 MIME 타입 리턴

### Q. ContentProvider URI의 주요 구성 요소는 무엇이며, ContentResolver는 데이터를 쿼리하거나 수정하기 위해 ContentProvider와 어떻게 상호 작용하나요?
- ContentProvider URI의 구성 요소는 아래와 같다
    - `content://` 스키마
    - ContentProvider를 식별하기 위한 권한
    - 가져오려는 데이터를 의미하는 Path
    - 경로 끝에 붙어 특정 데이터를 식별하는 ID
- ContentResolver와 ContentProvider간 상호작용 과정은 아래와 같다
    - contentResolver에서 query, insert, update, delete 메서드를 호출한다.
    - 안드로이드 시스템은 contentResolver에서 사용한 Uri의 권한 부분을 보고 그 권한에 해당하는 ContentProvider가 존재하는지 확인한 후 ContentProvider의 해당 메서드(query, insert, update, delete)를 호출한다.
    - ContentProvider에서 전달받은 URI의 path와 id를 분석하여 요청한 데이터가 무엇인지 파악한 후 데이터를 Cursor 객체의 형태로 ContentResolver에 반환한다.
    - ContentResolver에서 결과를 수신하여 처리한다.
