### 기본 동작

1. Activity 종료 및 재시작
    - 기존 Activity의 onPause→onStop→onDestroy메서드를 호출
    - 구성 변경이 완료된 이후 Activity의 onCreate 메서드가 호출
2. 리소스 다시 로드하기
    - 변경된 구성에 따라 레이아웃, 드로어블, 문자열과 같은 리소스를 다시 로드
3. 데이터 손실 방지
    - onSaveInstanceState 및 onRestoreInstanceState 메서드 혹은 ViewModel과 같은 다른 방식을 사용하여 기존 데이터를 복원

### 재생성을 유발하는 구성 변경

- 앱 디스플레이 크기 변경
- 앱 디스플레이 방향 변경
- 글꼴 크기 및 두꼐 변경
- 언어 변경
- 다크모드/라이트모드 전환
- 키보드 사용 가능 여부

### Activity 재생성 피하기

- Manifest파일에서 android:configChanges 속성을 추가할 경우 해당 속성에 선언된 구성 변경은 개발자가 수동적으로 처리한다고 시스템에 알리게 된다.
- 이 경우 andorid:configChanges에 선언한 구성 변경이 발생한 경우 기본 동작 대신, onConfigurationChanged 메서드가 호출되게 된다.
