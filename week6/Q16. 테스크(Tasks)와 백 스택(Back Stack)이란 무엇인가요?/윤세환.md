### Task
- 사용자가 특정 목적을 위해 상호작용하는 Activity의 집합
- 후입선출의 백스택 형식을 사용하여 Activity의 집합을 관리

### 백 스택
- 태스크 내 Activity의 기록을 유지
- 새 Activity가 생성될 떄 추가되며, 뒤로가기 클릭과 같이 Activity를 Task에서 제거하는 경우 pop하여 그 아래에 있던 Activity가 재개된다.

### 런치 모드
- Activity가 어떻게 인스턴스화되고 백 스택에서 어떻게 처리되는지를 결정
    1. standard
        - 기본 모드로, 인스턴스가 이미 백스택에 존재하더라도 새로운 인스턴스를 생성하여 사용한다.
    2. singleTop
        - 호출할 Activity가 현재 스택의 맨 위에 존재하는 경우 새 인스턴스를 생성하지 않고 기존 인스턴스를 사용한다.
        - 이 때 기존 인스턴스의 onNewIntent가 호출된다.
    3. singleTask
        - Activity가 새로운 태스크에서 실행되거나, 해당 Activity가 있는 Task로 이동하도록 한다.
        - 호출할 Activity가 백스택에 존재하는 경우 해당 인스턴스까지의 다른 Activity를 제거한 후 해당 인스턴스를 사용한다.
        - 태스크 내 해당 Activity 객체는 단 하나만 존재한다.
    4. singleInstance
        - Activity가 항상 새로운 Task를 생성하고 해당 Task에는 이 Activity 하나만 존재하도록 한다.
        - 이미 해당 Activity가 있는 Task가 존재한다면 새로운 Task를 만들지 않고 해당 Activity를 사용한다.
        - 시스템 상에 해당 Activity 객체는 단 하나만 존재한다.
    5. singleInstancePerTask
        - Activity가 항상 새로운 Task의 루트 Activity로 실행되도록 한다.
        - 하나의 Task내 해당 Activity가 단 하나만 존재한다.
        - singleInstance와는 달리, 인텐트 플래그로 `FLAG_ACTIVITY_MULTIPLE_TASK` 또는 `FLAG_ACTIVITY_NEW_DOCUMENT` 를 설정하여 Activity를 호출했다면 이 Activity를 가지고 있는 Task가 이미 존재하더라도 새로운 Task에 Activity를 생성하여 사용하므로 동일한 Activity 객체가 시스템에 2개 이상 존재할 수 있다.

### 인텐트 플래그
- Intent가 전송될 때 Activity가 시작되는 방식이나 백 스택의 동작을 수정하는데 사용
    - FLAG_ACTIVITY_NEW_TASK
    - FLAG_ACTIVITY_CLEAR_TOP
    - FLAG_ACTIVITY_SINGLE_TOP
    - FLAG_ACTIVITY_NO_HISTORY
- 개발자가 유동적으로 플래그를 설정할 수 있어, 동적으로 Activity처리 로직을 구현해야 하는 경우 사용한다.

### Q. singleTask와 singleInstance의 차이점은 무엇이며, 각각 어떤 시나리오에서 사용해야 하나요?
- singleTask의 경우 하나의 Task내 해당 Activity객체가 1개만 존재하는 구조로, 만약 해당 Activity가 이미 task에 존재하는 경우 해당 Task로 이동한 후 해당 Activity까지의 다른 Activity를 제거하여 해당 Activity를 사용한다.
- singleInstance의 경우 해당 Activity가 새로운 Task에서 수행하고, 그 Task에는 해당 Activity만 존재하도록 한다. 만약 해당 Activity가 이미 다른 Task에 존재하는 경우 해당 Task로 이동하게 되며, 시스템상에 해당 Activity객체가 단 1개만 존재하도록 한다.
