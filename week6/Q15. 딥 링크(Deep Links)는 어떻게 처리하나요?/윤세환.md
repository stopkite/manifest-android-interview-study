요약
1. 매니페스트에서 딥링크를 처리하고하 하는 Activity에 intent-filter로 딥링크 정보를 설정
2. 해당 Activity의 onCreate(특수한 경우 onNewIntent)에 인텐트로부터 Uri 형식의 딥링크 데이터를 가져와 처리한다.
3. adb 명령어를 사용하여 구현한 딥링크 처리 로직을 테스트한다.

---

### 1. 매니페스트에서 딥 링크 정의
- 딥링크를 처리하는 Activity에 대해, Manifest 파일에 딥링크 처리를 위한 intent filter를 설정해야 한다.
이 때 딥링크 URI는 아래 요소로 구성된다.
    - Scheme: URL 스키마
    - Host: 도메인
    - Path: URL의 경로

```xml
<activity android:name=".MyDeepLinkActivity" android:exported:"true">
	<intent-filter>
		<action android:name="android.intent.action.VIEW"/>
		<category android:name="android.intent.category.DEFAULT"/>
		<category android:name="android.intent.category.BROWSABLE"/>
		<!-- https://example.com/deepLink 라는 딥링크를 처리하고자 하는 경우 -->
		<data
			android:scheme="https"
			android:host="example.com"
			android.pathPrefix="/deepLink"/>
	</intent-filter>
</activity>
```

### 2. Activity에서 딥 링크 처리
- 해당 화면의 onCreate (launchMode에 따라서는 onNewIntent)에서 intent의 data로 전달된 딥링크 uri를 가져와 처리할 수 있다.

```kotlin
// https://example.com/deepLink?id=123 형식의 링크를 처리하는 경우
val intentData: Uri? = intent?.data
if (intentData != null) {
	val path: String? = intentData.getPath()
	val id: String? = intentData.getQueryParameter()
	navigateToFeature(path, id)
}
```

### 3. 딥 링크 테스트하기
- 아래 adb 명령어를 통해 딥 링크 처리  로직을 테스트할 수 있다.

```
adb shell am start -a android.intent.action.VIEW \
-d "https://example.com/deelLink?id=123" \
com.example.myapp # 앱 패키지 이름
```
