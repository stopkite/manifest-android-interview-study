### Bundle이란
- Activity, Fragment, Service와 같은 컴포넌트간 데이터를 공유하는데 사용되는 키-값 쌍 데이터 구조
- 일반적으로 안드로이드 앱 내부에서 작은 용량의 데이터를 효율적으로 전송하는데 사용
- 가볍고, 안드로이드 운영체제가 수비게 관리하고 전송할 수 있는 형식으로 데이터를 직렬화하도록 설계

### Bundle의 일반적인 사용 사례
- Activity간 데이터 전달
- Fragment간 데이터 전달
- 인스턴스 상태 저장 및 복원
    - Activity의 onSavedInstanceState, onRestoreInstanceState 등
- Service에 데이터 전달

### Bundle 동작 방식
- 데이터를 키-값 구조로 직렬화하는 방식으로 동작하며, 키는 문자열 형식 그리고 값은 기본 유형, Serializable, Parcelable 또는 다른 Bundle를 사용할 수 있다.

### Q. 구성 변경 중 onSaveInstanceState는 UI상태를 보존하기 위해 Bundle을 어떻게 활용하며, Bundle에 어떤 유형의 데이터를 담을 수 있나요?
- onSaveInstanceState에서 인자로 전달받은 Bundle에 putInt, putString과 같은 메서드를 사용하여 키-값의 형식으로 Bundle에 보존하고자 하는 데이터를 저장한다.
- 이후 onCreate 또는 onRestoreInstanceState에서 인자로 전달받은 bundle로부터 데이터를 가져와 복원한다.
- Bundle에는 기본 유형, Serializable, Parcelable 그리고 다른 Bundle 형식의 데이터를 저장할 수 있다.
