### 구성 변경의 종류
- 앱 디스플레이 크기 변경
- 앱 디스플레이 방향 변경
- 글꼴 크기 및 두꼐 변경
- 언어 변경
- 다크모드/라이트모드 전환
- 키보드 사용 가능 여부

### 구성 변경 처리 방법
- Activity의 onSaveInstanceState/onRestoreInstanceState를 통한 상태 저장 및 복원
- ViewModel을 사용한 데이터 유지
- onConfigurationChanged 메서드와 Manifest파일을 통한 수동 처리
- Compose에서의 rememberSaveable 사용

### 추가 팁
- Navigation 컴포넌트를 사용할 경우, Configuration Change가 발생해도 네비게이션 백스택은 유지된다.
- Configuration에 의존적인 값을 UI 레이어에 저장하지 않아야 한다.

### Q. 구성 변경(Configuration Change)을 처리하기 위한 전략에는 무엇이 있으며, ViewModel은 구성 변경으로부터 손실될 수 있는 UI 관련 데이터를 어떻게 보존하나요?
- 구성 변경 처리 전략에는 아래 방법들이 존재한다.
    - onSaveInstanceState를 통한 UI 데이터를 Bundle에 저장하는 방법
    - ViewModel을 사용하여 정보를 유지하는 방법
    - android:configChanges 태그를 사용하여 Activity가 특정 구성 변경을 onConfigurationChanged에서 처리하도록 하는 방법
- ViewModel의 경우, Activity/Fragment보다 더 생명주기가 긴 ViewModelStore에서 관리된다. 이 덕분에 Activity가 파괴된 후 다시 생성되더라도 처음 해당 Activity에서 생성한 ViewModel 객체는 유지되어 해당 객체를 그대로 사용할 수 있다.

### Q. AndroidManifest 파일에서 android:configChanges 속성은 Activity 생명주기와 동작에 어떤 영향을 미치며, Activity 재시작에 의존하는 것이 아니라 onConfigurationChanged() 메서드를 사용해야 하는 시나리오의 예시를 들어주세요.
- 일반적인 경우, 구성 변경이 발생한다면 `onPause -> onStop -> onDestroy -> onCreate -> onStart -> onResume` 의 생명주기를 따른다
- 만약 configChanges에 설정한 구성 변경이 발생한 경우에는, `onConfigurationChanged` 가 발생하며 해당 메서드에서 개발자가 직접 구성 변경에 대한 작업을 처리해야 한다.
- onConfigurationChanged 메서드를 사용해야 하는 예시로는
    - 영상 재생 중 화면 방향 변경이 감지된 경우
    - 영상통화, 드로잉 앱같은 실시간 동작이 수행될 때 화면 방향 변경/크기 변경이 발생한 경우
