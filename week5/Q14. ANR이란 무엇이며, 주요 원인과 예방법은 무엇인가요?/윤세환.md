### Application Not Responding
- 앱의 메인 스레드가 5초 이상 차단될 때 발생하는 안드로이스 시스템 에러
- 해당 에러 발생시 안드로이스 시스템은 사용자에게 앱을 종료할지, 대기할지를 선택할 수 있도록 안내한다.

### ANR 예방 작업
- 무거운 작업의 경우 메인 스레드가 아닌 다른 스레드에서 수행해야 한다.
- 데이터 동기화와 같이 백그라운드로에서 수행해야 하는 작업은 WorkManager를 사용한다.
- 리스트 데이터를 불러오는 경우에는 전체 데이터를 한번에 불러오는 대신 페이징을 사용하여 데이터를 더 작고 관리 가능한 청크로 가져와 UI 과부화를 방지한다.
- 구성 변경시의 UI 작업을 최소화한다.
- Android Studio의 Profiler를 사용하여 성능 및 메모리 누수를 모니터링한다.
- runBlocking과 같은 블로킹 호출을 피한다.
- 가벼운 지연 작업에는 Thread.sleep 대신 Handler.postDelayed를 사용하여 메인 스레드를 차단하지 않고 가벼운 지연 작업을 처리할 수 있다.
